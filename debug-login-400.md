# [OPEN] debug-login-400

## 症状
- H5 前端调用 `POST http://115.190.218.167/api/auth/login` 返回 `400 Bad Request`
- 浏览器控制台显示 `request.ts` 抛出 `HTTP 错误: 400`
- 同时出现 `:5174/favicon.ico 404`

## 当前上下文
- 前端默认后端地址已切到公网 API
- 后端公网健康检查已通过
- 后端注册/登录接口此前已通过 curl 验证

## 初始假设
- H1: 当前登录使用的账号并不存在于公网 ECS 上的 SQLite 数据库里
- H2: 当前登录密码错误，后端业务校验返回 400
- H3: 前端发出的登录请求体字段格式不符合后端 `LoginRequest` 预期
- H4: 前端命中的并不是你以为的那套运行时地址，例如本地存储中的 `dev_api_base_url` / `dev_mock_mode` 干扰
- H5: `favicon.ico 404` 与登录失败无关，只是静态资源缺失噪音

## 计划
- 先收集前端实际请求体、运行时 API 地址、mock 状态
- 再对照后端返回体判断是业务拒绝还是协议错误
- 若确认根因，再做最小修复
