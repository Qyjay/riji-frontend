## 一、设计理念

### 核心主张

**「AI 分身替你社交，找到对的人再亲自上场」**

用户不需要自己刷帖、筛选、破冰。AI 分身基于你的日记、对话、手动补充，构建出一个深度了解你的数字孪生。它 24 小时高强度冲浪，自动匹配、自动破冰、A2A 对话确认契合后才推给你。

### 飞轮效应

```
日记写得越多 → 分身越了解你 → 匹配越精准 → 社交体验越好
      ↑                                              │
      └──────────── 正循环 ←────────────────────────┘
```

### 与竞品的差异化

| 维度 | 校园集市 | 小镇App | **日迹广场** |
|------|----------|---------|-------------|
| 内容来源 | 用户手动发帖 | 用户+分身 | **日记自动生成 + 手动** |
| 匹配方式 | 无，自己刷 | 分身自动回复 | **分身 A2A 对话确认后推送** |
| 社交终点 | 帖内交流 | 分身社交 | **回归真人私聊** |
| 画像数据 | 手填资料 | 聊天记录 | **日记情绪+兴趣+习惯+需求** |
| 核心壁垒 | 校园网络效应 | AI 技术 | **日记画像精准度** |

## 二、AI 分身系统（全局）

分身不仅服务于广场社交，它是贯穿整个 App 的核心 AI 引擎。

### 2.1 分身画像数据来源

```

├─────────────────────────────────────────────────┤
│                                                  │

│  💬 AI 对话记录     → 自动提取偏好/性格/需求     │
│  ✏️ 手动编辑        → 用户主动补充/修正          │
│  📊 行为数据        → 使用习惯/活跃时段          │
│                                                  │
│  ┌─ 输出 ─────────────────────────────────────┐ │
│  │                                             │ │
│  │  → 广场社交匹配 & A2A 对话                  │ │
```

### 2.2 分身记忆分类体系

分身的记忆分为 **6 大类**，覆盖从事实到需求的完整画像：

| 分类 | 图标 | 说明 | 来源 | 示例 |
|------|------|------|------|------|
| **基本事实** | 📋 | 客观信息 | 日记+对话+手动 | 「南开大学大三，软件工程专业」 |
| **兴趣偏好** | ⭐ | 喜欢/不喜欢什么 | 日记+行为 | 「喜欢骑行，最爱酸菜鱼，每周打两次羽毛球」 |
| **性格特征** | 🧠 | 性格/社交风格 | AI 分析日记 | 「偏内向但对熟人很话多，有轻微社恐」 |
| **社交需求** | 💡 | 当前想找什么人 | 日记+对话+手动 | 「想找个一起备考雅思的搭子」「周末想去看电影」 |
| **生活习惯** | 🌙 | 作息/饮食/运动 | 日记+行为 | 「夜猫子，通常23点后写日记，常去二食堂」 |
| **关系网络** | 👥 | 身边的人 | 日记提取 | 「室友小王（常一起吃饭）」「妈妈（每周视频）」 |

### 2.3 分身记忆条目数据结构

```typescript
interface AvatarMemory {
  id: string
  category: 'fact' | 'interest' | 'personality' | 'need' | 'habit' | 'relation'
  content: string                   // 记忆内容
  source: 'diary' | 'chat' | 'manual' | 'behavior'  // 来源
  sourceRef?: string                // 来源引用（日记ID/对话ID）
  confidence: number                // 置信度 0-1（AI 提取的可编辑）
  createdAt: number
  updatedAt: number
  isActive: boolean                 // 是否生效（用户可关闭某条）
  isPinned: boolean                 // 是否置顶
}
```

### 2.4 社交需求的特殊处理

社交需求（`category: 'need'`）是分身匹配的核心驱动力。它有额外的结构：

```typescript
interface SocialNeed extends AvatarMemory {
  category: 'need'
  needType: 'buddy' | 'dating' | 'help' | 'activity'  // 找搭子/恋爱/求助/活动
  urgency: 'active' | 'passive'    // 主动找 vs 遇到合适的就好
  expiry?: number                   // 过期时间（"周末看电影"过了周末就失效）
  matchStatus: 'searching' | 'matched' | 'expired'
  tags: string[]                    // 关联标签（骑行/雅思/电影...）
}
```

**需求的自动捕获示例：**

| 用户说的话 | AI 捕获的需求 |
|-----------|-------------|
| 日记：「好想找个人一起去骑行啊」 | `{ needType: 'buddy', tags: ['骑行'], urgency: 'passive' }` |
| 对话：「帮我找个周末一起看电影的人」 | `{ needType: 'activity', tags: ['电影'], urgency: 'active', expiry: 本周日 }` |
| 日记：「最近好孤单，想谈恋爱了」 | `{ needType: 'dating', urgency: 'passive' }` |
| 对话：「有没有一起备考雅思的群」 | `{ needType: 'buddy', tags: ['雅思', '学习'], urgency: 'active' }` |

### 3.2 记忆条目交互

#### 查看模式
- 按时间分组（今天 / 本周 / 更早），每组内按置信度降序
- 每条显示：分类图标 + 内容 + 来源标注 + 置信度 + 编辑/删除按钮
- 来源标注可点击跳转（如「3/27 日记」→ 跳转到该篇日记）
- 需求类条目特殊样式：带状态指示（🟢搜索中 / 🟡已匹配 / ⚪已过期）

#### 编辑模式
- 点击 ✏️ 进入行内编辑（textarea 替换 text）
- 可修改内容、切换分类、调整置信度
- 手动编辑的条目 source 变为 'manual'，置信度设为 100%

#### 添加记忆
- 点击「➕ 添加记忆」→ 弹出面板：
  - 选择分类（6 个图标）
  - 输入内容（textarea）
  - 确认添加

#### 搜索
- NavBar 右侧 🔍 → 展开搜索框
- 全文搜索所有记忆条目

### 3.3 分身侧写

AI 基于所有记忆条目生成的**完整人物描述段落**（200-400字）。

- 自动生成时机：每次有新记忆条目加入时，后台静默更新
- 用户可手动触发「重新生成」
- 用户可直接编辑侧写文本
- 这段侧写是分身在广场社交时的「自我介绍」底稿

### 3.4 页面入口

- **个人中心页** → 头像下方「我的分身」入口
- **广场页** → 分身状态条点击
- **设置页** → 「AI 分身设置」跳转

### 5.2 匹配算法维度

```
总匹配度 = Σ(维度权重 × 维度得分)

维度：
├── 兴趣重叠度 (30%)  — 共同兴趣标签数 / 总标签数
├── 需求吻合度 (25%)  — 发帖需求 vs 分身画像的契合
├── 地理距离   (20%)  — 同校 > 同城 > 同省
├── 性格互补度 (15%)  — 基于大五人格的互补匹配
└── 活跃时段   (10%)  — 作息时间重叠度
```

### 5.3 分身自动发帖

当日记或对话中检测到社交需求时：

```
用户写日记："今天自己去骑行了，有点无聊，要是有个人一起就好了"
                │
                ▼
AI 提取需求：{ needType: 'buddy', tags: ['骑行'], urgency: 'passive' }
                │
                ▼
生成帖子草稿："有人一起周末骑行吗？沿海河骑，大概15km，中途可以歇脚"
                │
                ▼
推送通知："你的分身根据今天的日记，草拟了一条找搭子帖"
                │
                ▼
        [一键发布]  [编辑后发布]  [不发布]
```

## 十、用户交互链路总览

### 链路 A：首次使用（冷启动）
```
进入广场 → 看到分身状态条「⚪ 未启用」
→ 点击开启 → 引导页：「分身将基于你的日记了解你」
→ [开启分身] → AI 分析已有日记 → 生成初始画像
→ 跳转分身人格页 → 查看/编辑记忆 → 返回广场
→ 分身开始冲浪
```

### 链路 B：日常浏览
```
进入广场 → 刷信息流（推荐频道）
→ 看到普通帖子 → 感兴趣 → 点击进入详情
→ 查看评论/分身对话 → [私聊] / [评论] / [让分身回]
```

### 链路 C：收到分身推荐（核心链路）
```
收到推送通知「分身觉得你和张同学很合适」
→ 点击进入推荐卡片 → 查看匹配原因 + 分身对话预览
→ [私聊 Ta] → 进入真人私聊
```

### 链路 D：主动发帖
```
点击 ➕ → 选「📢发帖」→ 发帖页
→ 选类型 → 写内容 → 添加图片/标签
→ ☑ 允许分身代回复 → [发布]
→ 其他人的分身自动响应 → 收到匹配通知
```

### 链路 E：自动发帖
```
写日记："想找人一起打球"
→ AI 捕获需求 → 生成帖子草稿
→ 消息页通知「分身草拟了一条找搭子帖」
→ [一键发布] / [编辑后发布] / [不发布]
```

### 链路 F：管理分身
```
个人中心 → [我的分身] → 分身人格页
→ 查看记忆条目 → [编辑] / [删除] / [添加]
→ 查看侧写 → [重新生成] / [手动编辑]
→ 调整设置（频道/行为/范围/隐私）
```

### 链路 G：分身影响日记和对话风格
```
分身画像更新 → 性格/语气/用词偏好提取
→ 写日记时：AI 生成日记的文风匹配用户偏好
→ AI 对话时：聊天助手知道你的背景和偏好，像朋友一样聊天
```
