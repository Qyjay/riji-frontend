## 一、产品设计理念

### 1.1 为什么做日迹？

大学生群体面临一个悖论：**生活最丰富的四年，却是最容易被遗忘的四年。**

课堂笔记、社团活动、深夜食堂、考试周焦虑、毕业季离别——这些碎片化的日常构成了大学最珍贵的记忆，但绝大多数人没有记录的习惯。原因很简单：

- **太忙了**——课程、社团、实习多线并行，没时间写日记
- **太麻烦了**——传统日记 App 要求结构化输入，心理门槛高
- **写不好**——想记录但不知道怎么表达，写出来觉得流水账

日迹的回答是：**你不需要会写日记，你只需要记录此刻。**

拍一张照片、说两句话、敲几个字——AI 帮你把碎片拼成日记，把日记变成漫画、小说、分享卡片。记录是零压力的，产出是有惊喜的。

### 1.2 核心价值主张

**「让每一天都值得被记住」**

日迹不是一个工具，是一个**伙伴**。它的设计哲学可以用三个关键词概括：

| 关键词 | 含义 | 在产品中的体现 |
|--------|------|----------------|
| **轻量** | 记录不应该是负担 | 快拍2步完成、语音按住说话、AI 自动生成日记 |
| **温暖** | 产品应该让人觉得被关心 | 手绘风格、情绪追踪、每日运势、纪念日提醒 |
| **创造** | 记录不只是存储，是创作 | 日记→漫画、日记→小说、日记→分享卡片 |

### 1.3 目标用户画像

- **年龄：** 18-24 岁大学生
- **特征：** 生活节奏快、社交活跃、对 AI 接受度高、审美有要求
- **痛点：** 想记录生活但觉得麻烦，想表达但词穷，想分享但没有好看的形式
- **使用场景：** 课间拍下老师的板书、食堂遇到好吃的、跑步后的自拍、深夜的胡思乱想

## 二、视觉设计体系（Design System）

### 2.1 Doodle 手绘风格——设计哲学

日迹选择 **Doodle 手绘涂鸦风** 作为核心视觉语言，这不是随意的选择，而是基于三个深思熟虑的理由：

**① 降低记录压力**

精致的 UI 会给用户一种「这是一个正式的工具」的心理暗示。手绘风格的不完美感传递的信息是：「这里没有对错，随便写写就好。」研究表明，涂鸦风格的界面能降低用户的心理门槛，鼓励更自由的表达。

**② 温暖亲切**

大学生的生活已经充满了冰冷的工具——教务系统、考试平台、求职网站。日迹应该是一个让人感到温暖的角落。手绘线条、暖色调、不规则圆角——每一个细节都在说「这里很安全，放松下来。」

**③ 差异化竞争**

市面上的日记类 App 几乎清一色的 Material Design 或 iOS 原生风格。Doodle 手绘风让日迹在第一眼就与众不同，建立强烈的品牌辨识度。

### 2.3 手绘图标系统（40+ 纯手写 SVG）

日迹没有使用任何图标库（无 IconFont、无 Material Icons、无 Feather）。所有图标均为**纯手写 SVG path**，由 `DoodleIcon.vue` 组件统一渲染。

**为什么自己画？**

- 市面图标库的线条过于规整，与 Doodle 风格格格不入
- 手写 SVG 的线条带有自然的弧度和粗细变化，与整体风格完美融合
- 40+ 图标覆盖全部功能场景，体现产品匠心

**技术实现：**

```vue
<!-- 使用方式 -->
<DoodleIcon name="book" color="#E8855A" :size="48" />
```

组件内部通过 `name` 映射到对应的 SVG path 数据，支持自定义 `color`、`size`（rpx 单位），并可选通过 SVG `feTurbulence` 滤镜叠加手绘抖动效果。

### 2.4 手绘抖动效果（feTurbulence）

通过 SVG 滤镜在图标和元素上叠加细微的噪声位移，模拟手绘笔触的不规则感：

```xml
<filter id="n">
  <feTurbulence type="fractalNoise" baseFrequency="0.9"
                numOctaves="4" stitchTiles="stitch" />
  <feColorMatrix type="saturate" values="0" />
</filter>
```

此滤镜应用于页面背景，产生细腻的纸张纹理，增强「在纸上涂写」的沉浸感。

### 2.5 不规则圆角

传统 UI 的 `border-radius` 四个值相同（如 `16px`），日迹使用四值不同的圆角模拟手绘矩形的不对称感：

```scss
// 四种预设变体，交替使用避免单调
.doodle-box    { border-radius: 28rpx 36rpx 24rpx 40rpx; }
.doodle-box-v2 { border-radius: 32rpx 24rpx 36rpx 28rpx; }
.doodle-box-v3 { border-radius: 24rpx 32rpx 28rpx 36rpx; }
.doodle-box-v4 { border-radius: 36rpx 28rpx 32rpx 24rpx; }
```

### 2.6 手写字体

标题和装饰文字使用 **ZCOOL KuaiLe（站酷快乐体）** 网络字体，在保持可读性的同时增加手写感。通过 `.font-handwrite` 工具类按需应用，正文保持系统默认字体以确保阅读舒适度。

### 2.7 Design Token 体系

所有设计变量统一在 `src/common/design-tokens.scss` 中管理：

| Token 类别 | 示例 | 作用 |
|------------|------|------|
| 颜色 | `$primary`, `$text-title` | 保证全局色彩一致 |
| 圆角 | `$radius-sm` (16rpx), `$radius-card` (32rpx) | 统一圆角规格 |
| 间距 | `$space-sm` (16rpx) ~ `$space-xl` (64rpx) | 8pt 网格系统 |
| 模块色 | `$color-diary`, `$color-study` ... | 功能区分色 |

## 三、全功能用户交互链路

### 3.1 素材收集（写日记）— 双路径设计

这是日迹最核心的功能入口，也是经过深度思考后重新设计的交互链路。

#### 设计洞察

记录生活素材只有**两种心智模型**：

1. **「我看到了什么」** → 视觉先行，拍照为主，顺手说两句
2. **「我想说什么」** → 文字先行，写或者说，图片可选

传统设计按「输入方式」分类（拍照/相册/文字/语音），但用户真正的决策依据是**「我现在想记什么」**。

#### 交互流程

```
点击底部 TabBar 中间按钮
         │
         ▼
  ┌─────────────────────┐
  │    记录此刻          │
  │                      │
  │  📷 快拍    📝 随记   │
  │  拍下此刻   写点什么   │
  │  + 说两句             │
  └─────────────────────┘
         │
    ┌────┴────┐
    ▼         ▼
  快拍路径    随记路径
```

**路径 A：快拍（Camera-first，最快 2 步完成记录）**

```
点「快拍」→ 直接打开相机
                │
                ▼
         拍照 / 选照片
                │
                ▼
  ┌──────────────────────┐
  │     [照片大图预览]      │
  │                        │
  │  说点什么...       🎤   │  ← 可选：打字或按住录音转文字
  │                        │
  │      [✓ 记下了]         │
  └──────────────────────┘
                │
                ▼
         保存 → 返回首页
```

- 相机**立即弹出**，无中间页面
- 拍完进入极简卡片：照片 + 一行输入 + 语音按钮
- 🎤 按住录音，松开自动转文字填入
- 一屏完成，不滚动
- **最快 2 步**：快拍 → 拍照 → 记下了

**路径 B：随记（Text-first，语音无缝融入）**

```
点「随记」→ 进入文字记录页
                │
                ▼
  ┌──────────────────────┐
  │  写下此刻的想法...       │  ← textarea 自动聚焦
  │  (光标闪烁，键盘弹出)    │
  │                        │
  │  [照片区域 + 添加按钮]   │
  │  📍 位置信息             │
  │                        │
  │  ── 今日已记录 N 条 ──   │
  │  [素材列表]              │
  └──────────────────────┘
  ┌──────────────────────┐
  │ 📷 拍照 │ 🎤 按住说话 │ ✓ 保存 │  ← 底部工具栏
  └──────────────────────┘
```

- textarea 进入即自动聚焦，键盘弹出，可直接打字
- 🎤 采用**按住说话**模式：按下开始录音，松开自动转文字追加到 textarea
- 无需切换到额外的录音弹层，语音输入如同微信语音输入法一样自然
- 可随时添加照片辅助

#### 操作步数对比

| 场景 | 旧设计 | 新设计 | 减少 |
|------|--------|--------|------|
| 快速拍照记录 | 4步（按钮→选项→页面→📷） | 2步（按钮→快拍→拍） | **50%** |
| 拍照 + 说两句 | 6步 | 3步（拍→说→保存） | **50%** |
| 纯写文字 | 3步（按钮→选项→页面） | 2步（按钮→随记→直接打字） | **33%** |
| 语音转文字 | 5步（按钮→选项→页面→🎤→弹层） | 2步（按钮→随记→按住说） | **60%** |

#### 为什么这样设计更好？

1. **减少选择焦虑**：2 个选项 vs 4 个选项，决策时间更短
2. **意图匹配**：按用户心智模型分类，而非输入方式
3. **操作步数最少化**：核心动作（记录）的步数减半
4. **语音零摩擦**：按住说话融入工具栏，无需独立弹层

### 3.2 AI 日记生成

```
今日素材（照片/文字/语音）
         │
         ▼
    AI 自动分析
   （情绪提取 + 内容理解）
         │
         ▼
  生成日记初稿（带文风引擎）
         │
         ▼
  ┌──────────────────────┐
  │  [日记预览页]            │
  │                        │
  │  2026年3月31日 周二      │
  │  今天在食堂发现了...      │
  │                        │
  │  [编辑] [重新生成] [满意]  │  ← 可修改（限次）
  └──────────────────────┘
         │
         ▼
  保存到日记列表 + 情绪趋势图更新
```

**设计亮点：**
- AI 不是替代用户写日记，而是**帮助用户把碎片变成叙事**
- 支持文风引擎自定义：文艺/幽默/简洁/温暖/中二/古风/科幻/电影
- 用户可修改，但限制次数，鼓励「先接受再打磨」而非「反复纠结」

### 3.3 衍生创作

```
一篇日记
    │
    ├──→ 🎨 一键生成漫画（AI 图像）
    ├──→ 📖 生成小说章节（AI 文本）
    └──→ 🃏 生成分享卡片（模板渲染）
```

**设计理念：** 日记不只是记录，更是创作的起点。AI 让「从日记到作品」的成本趋近于零，激发用户的创作欲。

### 3.4 AI 对话

```
底部 Tab「消息」→ AI 对话页
         │
         ▼
  ┌──────────────────────────────┐
  │  [AI 伙伴信息卡]                │
  │  [对话气泡列表]                  │
  │                                │
  │  用户：今天好累啊                │
  │  AI：（流式打字机输出）           │
  │                                │
  │  [快捷操作栏] 记录素材/看运势/... │
  │  [附件预览区] 📷📎（可删除）      │
  │  [录音状态条] 🔴 录音中... 3s    │
  │  [📎] [输入框] [🎤] [发送]       │
  └──────────────────────────────┘
```

**设计亮点：**
- **流式打字机效果**：AI 回复逐字出现，模拟真人打字的节奏感
- **生活助手定位**：不是通用 ChatGPT，而是了解你生活的伙伴
- 基于用户的日记、情绪数据提供个性化回复
- **输入栏非 fixed 布局**：避免 App 端键盘弹出时输入框贴键盘，使用 `cursor-spacing` 控制间距
- **流式滚动优化**：AI 输出时用 `scroll-top` 代替 `scroll-into-view`，关闭动画防止抖动

**语音输入（转文字）：**
- 点击🎤按钮开始录音，再点停止（最长60秒）
- 录音时显示状态条：红色脉冲点 + 录音时长 + 取消按钮
- 录音结束后上传后端 `POST /api/materials/voice`，后端 ASR 转写
- 转写文字自动填入输入框，可继续编辑后发送

**附件上传：**
- 点击📎按钮弹出三选一：拍照 / 从相册选择 / 选择文件
- 图片最多9张（压缩上传），文件支持 pdf/ppt/doc/xls/txt 等
- 已选附件在输入栏上方预览区横滑展示，可单独删除
- 发送时附件信息附加到消息中传给 AI

**对话自动转素材：**
- 用户与 AI 聊天后，对话按静默阈值（默认30分钟，可在设置页调整）自动切分为对话段
- 每段对话由 AI 自动生成标题、摘要、情绪标签，创建为 `type="chat"` 的素材
- 💬 对话素材在素材时间线中与 📷 照片、✍️ 文字素材平级展示
- 点击「展开对话 ›」可查看原始对话记录（Bottom Sheet 只读展示）
- 所有类型素材统一参与每日 AI 日记生成
- 设置页可控制：总开关、静默阈值、最小轮数、toast 提示

### 3.5 情绪追踪

```
首页 → 情绪日历入口
         │
         ▼
  ┌──────────────────────┐
  │  [月历视图]              │
  │  每天一个情绪 emoji       │
  │  😊😐😢😊😊😴😊...       │
  │                        │
  │  [情绪趋势折线图]         │
  │  本周情绪：偏积极 ↗       │
  └──────────────────────┘
```

**设计理念：** 情绪是自动提取的（基于日记内容 AI 分析），用户无需手动标记。降低使用门槛的同时，提供有价值的自我认知。

### 3.6 社交匹配

```
发现页 → 搭子匹配
         │
         ▼
  AI 分析用户画像（兴趣、作息、性格）
         │
         ▼
  推荐匹配对象 + 匹配度报告
         │
         ▼
  发送搭子申请 → 对方同意 → 开始聊天
```

**设计理念：** 大学生社交核心需求是「找到同类」。AI 基于用户的日记内容、行为数据生成画像，比传统问卷匹配更准确、更有趣。

### 3.7 成长轨迹

```
个人中心 → 成长轨迹
         │
         ├──→ 📊 写作统计（字数/频率/连续天数）
         ├──→ 🏆 成就系统（解锁成就徽章）
         └──→ 📋 学期报告（AI 生成学期总结）
```

**设计理念：** 游戏化机制（成就系统）+ 数据可视化（写作统计）+ AI 洞察（学期报告），三管齐下驱动长期留存。

### 3.8 纪念日管理

```
发现页 → 纪念日
         │
         ├──→ 手动添加纪念日
         ├──→ AI 自动从日记提取（识别重要事件）
         └──→ 那年今日回忆（到期自动推送）
```

**设计亮点：** AI 自动提取纪念日——用户在日记里写了「今天和小明确定关系了」，AI 会自动识别并创建纪念日，无需手动操作。

### 3.9 每日运势

```
首页功能入口 → 每日运势
         │
         ▼
  AI 生成个性化运势（学业/社交/健康）
  结合用户近期日记情绪生成建议
```

### 3.10 文风引擎

```
设置 → AI 写作偏好
         │
         ├──→ 选择写作风格（文艺/幽默/简洁/温暖/中二/古风/科幻/电影）
         ├──→ 多选风格标签（善用比喻/多用细节/情感充沛/简短干练...）
         └──→ 自定义提示词（200字以内自由描述）
```

**设计理念：** 每个人表达风格不同。文风引擎让 AI 生成的日记「听起来像你自己写的」，而不是千篇一律的 AI 味。

### 3.11 对话自动转素材

```
用户与 AI 聊天
      │
      ▼
静默超过阈值（默认30分钟）/ 用户离开聊天页
      │
      ▼
后端自动封闭对话段 → AI 提取标题/摘要/情绪/标签 → 生成 chat 素材
      │
      ▼
素材时间线新增 💬 卡片（可展开查看原始对话）
      │
      ▼
每日汇总时，与照片/文字素材一起参与 AI 日记生成
```

**设计亮点：**
- **零操作**：用户正常聊天即可，素材生成完全自动
- **可控**：设置页可关闭、调整阈值、控制最小轮数
- **轻提示**：可选 toast「已记录为今日素材 ✓」，1.5秒消失，不打断聊天
- **可追溯**：素材卡片支持展开查看原始对话记录

### 3.12 广场搜索与刷新

```
广场页（发现页）
      │
      ├── 频道 Tab 栏（推荐 / 找搭子 / 求助 / 恋爱 / 分享）
      ├── 分身状态条（冲浪中 · 已浏览N条 · 匹配N条）
      ├── 🔍 搜索栏（输入关键词，页内实时筛选）
      ├── 📦 工具箱（折叠区，默认收起）
      └── 帖子信息流（支持下拉刷新 + 上拉加载更多）
```

**设计亮点：**
- **页内搜索**：搜索框在「冲浪中」状态条下方，输入关键词后 300ms 防抖自动筛选，不跳转页面
- **搜索范围**：匹配帖子正文、话题标签、作者昵称、位置
- **搜索 + 频道可组合**：选「找搭子」频道后搜索「骑行」，只返回找搭子类型中含「骑行」的帖子
- **下拉刷新**：利用 scroll-view 原生 refresher，重置分页、重新拉帖子和分身状态
- **工具箱上移**：从信息流底部移至搜索栏下方，用户进页面就能看到，默认折叠不占空间

## 四、交互设计原则

### 4.1 减少认知负荷

| 设计决策 | 减负方式 |
|----------|----------|
| 双路径选择（快拍/随记） | 2 个选项 vs 4 个，决策时间减半 |
| 情绪自动提取 | 无需手动标记，AI 从文本分析 |
| 纪念日自动识别 | 无需手动添加，AI 从日记提取 |
| 文风引擎预设 | 8 种风格一键选择，不需要自己描述 |

### 4.2 快速完成核心动作

日迹的核心动作是**记录素材**。所有设计决策都服务于一个目标：**让记录的步数最少化。**

- 快拍路径：2 步完成
- 语音输入：按住即说，无需切换模式
- 照片记录：拍完自动进入编辑，不需要再找入口

### 4.3 情感化设计

- **手绘风格**：温暖、不完美、有人情味
- **提示语设计**：「今天还没有记录，快来添加第一条吧 ✨」而非冰冷的「暂无数据」
- **成就反馈**：完成记录时的 `✓ 已记录` toast + 成就徽章解锁
- **色彩心理**：暖杏橙主色传递温暖与活力，奶油白背景营造纸张质感

### 4.4 渐进式发现

日迹有 10+ 个功能模块，但不会一次性暴露给用户：

- **底部 Tab（5个）** ：日记、发现、写、消息、我的——覆盖最高频场景
- **首页功能网格** ：展示 6-8 个常用功能入口
- **侧边栏抽屉** ：收纳更多功能，不干扰主界面
- **发现页** ：社交、纪念日等低频功能入口

用户不会被功能淹没，而是在使用过程中自然发现更多能力。

```
  │      │     │     │      └─ 学期报告
  │      │     │     │

  ├─ 日记详情
  ├─ 情绪日历
  ├─ 衍生内容（漫画/小说/卡片）
  └─ 文风引擎
```

### 5.2 页面层级

| 层级 | 页面 | 导航方式 |
|------|------|----------|
| L0 — Tab 页 | 日记、发现、消息、我的 | `uni.redirectTo`（Tab 间切换，不堆栈） |
| L1 — 功能页 | 日记详情、快拍、随记、对话、纪念日... | `uni.navigateTo`（压栈，可返回） |
| L2 — 详情页 | 漫画预览、小说阅读器、分享卡片... | `uni.navigateTo` |

### 5.3 导航策略

- **Tab 切换用 `redirectTo`**：避免页面堆栈过深，保证 Tab 间切换流畅
- **功能页用 `navigateTo`**：保留返回路径，符合用户心理模型
- **所有页面 `navigationStyle: custom`**：统一使用 `CustomNavBar` 组件，保证手绘风格一致性
- **scroll-view 精确计算**：所有滚动区域高度通过 JS 动态计算（`windowHeight - navHeight - tabBarHeight`），避免 iOS/Android 差异

## 六、技术亮点

### 6.1 Mock / 真实 API 无缝切换

```typescript
// services/config.ts
export let USE_MOCK: boolean = _readMock()
export function setMockMode(val: boolean) {
  USE_MOCK = val
  uni.setStorageSync('dev_mock_mode', val)
}

// services/api/diary.ts
export async function getDiaries(page, pageSize) {
  if (USE_MOCK) return mock.getDiaries(page, pageSize)  // Mock 数据
  return request({ url: '/diaries', ... })                // 真实 API
}
```

**设计价值：**
- 开发者选项中一键切换 Mock 模式，**无需后端即可完整体验所有功能**
- 评委和用户可以立即看到完整的产品效果
- 8 个 API 模块均有完整的 Mock 数据覆盖
- 使用 ES Module `export let` + live binding 实现运行时动态切换，无需重启应用

### 6.2 纯手写 SVG 图标系统

`DoodleIcon.vue` 组件封装了 40+ 纯手写 SVG 图标：

- 统一 API：`<DoodleIcon name="book" color="#E8855A" :size="48" />`
- 支持颜色动态切换（选中/未选中态）
- 支持 SVG feTurbulence 滤镜叠加
- 零依赖，无需加载外部图标库，首屏渲染更快

### 6.3 响应式布局（rpx 体系）

所有布局尺寸使用 UniApp 的 rpx 单位（750rpx = 屏幕宽度），确保：

- iPhone SE 到 iPad Pro 全尺寸适配
- H5 / 小程序 / App 三端一致的视觉效果
- `border` 和 `box-shadow` 保留 px，避免亚像素模糊

### 6.4 状态管理与持久化

使用 Pinia + `pinia-plugin-persistedstate`，关键状态（用户设置、主题、提醒时间等）自动持久化到 uni.storage：

```typescript
const useSettingsStore = defineStore('settings', () => { ... }, {
  persist: { storage: uniStorage }
})
```

用户关闭 App 后重新打开，所有偏好设置自动恢复。

### 6.5 手绘动画系统

`src/common/animations.scss` + `handdrawn.scss` 提供：

- **圈圈动画**：情绪选中时的手绘圆圈绘制动画（`stroke-dashoffset` 渐变）
- **波浪下划线**：Section 标题的手绘波浪装饰
- **按压反馈**：`.press-feedback` 全局工具类，统一触摸反馈
- **脉冲动画**：中间写日记按钮的呼吸灯效果

## 七、竞品差异化

### 7.1 与主流日记 App 对比

| 维度 | 传统日记 App | 日迹 |
|------|-------------|------|
| **记录方式** | 打开 → 写字 → 保存 | 拍照/说话/打字 → AI 生成日记 |
| **创作能力** | 无 | 日记→漫画/小说/分享卡片 |
| **社交功能** | 无或简单分享 | AI 画像匹配 + 搭子社交 |
| **情绪追踪** | 手动标记或无 | AI 自动提取，零操作 |
| **视觉风格** | Material / iOS 原生 | Doodle 手绘涂鸦风，强辨识度 |
| **目标用户** | 全年龄段 | 聚焦大学生，功能量身定制 |
| **AI 深度** | 无或浅层 | 贯穿全链路：生成/润色/画像/匹配/运势 |

### 7.2 为什么大学生会选择日迹？

1. **零门槛**：不需要会写日记，拍照说话就行
2. **有惊喜**：AI 帮你把流水账变成优美的文字，日记还能变漫画
3. **有社交**：通过日记和生活方式找到同频的伙伴
4. **有温度**：手绘风格不是冰冷的工具，是暖暖的日常角落
5. **有成长**：情绪追踪、成就系统、学期报告让你看到自己的变化
